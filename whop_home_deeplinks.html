Android testing (develop build)

Build: com.whop.whopapp.develop  ·  signed in as @deeppandya91.

Browser tap. Android now auto-verifies the /home-feed path (added to the manifest in this change), so a tapped whop.com/home-feed… link opens the app. The /joined/home-feed… alias also works (the parser strips joined). On the develop build, verification isn't automatic — enable it once in Settings → Apps → Whop (develop) → Open by default → Open supported links → add whop.com.

adb (always works, bypasses link verification). Pushes the URL as the deeplink extra. The URL is single-quoted inside the device command so an unquoted & isn't eaten by the phone's shell (which would truncate the link before &a=…):

adb shell "am start -n com.whop.whopapp.develop/com.whop.whopapp.MainActivity \
  --es deeplink '<URL>'"

Android-specific behavior notes

  • The parser resolves /home-feed before the bot-route / experience parsers, so home-feed isn't mistaken for a company route.
  • Bare /home-feed navigates to the Home tab (Main(home)); the experience variant runs navigateSequence(Main(home), Experience(...)) so the Home tab is selected underneath.
  • A deeplink launch suppresses the townhall feed load and the rating prompt, so it opens straight to the target.
  • Deeplinks delivered while the app is backgrounded/cold are replayed once the UI is ready.
