Raffle Business — API Flow

Mapping the raffle seller journey onto Whop's existing public API (@whop/sdk / /api/v1), plus the gaps that need new endpoints.

Journey → API mapping

1 Create account on Whop

const account = await whop.accounts.create({
  email: "jack@example.com",
});
// → account.id  (biz_xxx)
// Self-serve sellers authenticate via OAuth instead; this is the platform path.

2 Start a raffle business

const company = await whop.companies.create({
  email: "jack@example.com",
  title: "Jack's Raffles",
});
// → company.id  (biz_xxx)  = your raffle business

3 What item do you want to put up for raffle?

The picker — Pokémon cards / Ebike / Robux giftcard / Socks / Labubus — just pre-fills the title.

const raffle = await whop.products.create({
  company_id: company.id,
  title: "Charizard PSA 10 — Raffle",
  headline: "1,000 tickets · winner drawn live",
  collect_shipping_address: true,   // physical prizes; false for Robux (digital)
});
// → raffle.id  (prod_xxx)

4 Prompt to design product page (upload pic of your item)

const file = await whop.files.create({
  filename: "charizard-psa10.png",
  visibility: "public",
});
// PUT the image bytes to file.upload_url with file.upload_headers
// Poll whop.files.retrieve(file.id) until upload_status === "ready"
// → file.id
The AI "design my product page" prompt is not a public endpoint yet. For now: upload a photo of your actual item.

5 Choose raffle min ticket $ amount

const ticket = await whop.plans.create({
  company_id: company.id,
  product_id: raffle.id,
  plan_type: "one_time",
  initial_price: 5.0,               // ← min ticket $ amount
  currency: "usd",
  title: "Raffle Ticket",
  stock: 1000,                      // total tickets available (optional)
  image: { id: file.id },           // ← prize photo from step 4
});
// → ticket.id  (plan_xxx)  +  ticket.purchase_url

6 Choose raffle deadline

No real deadline primitive exists. expiration_days is membership expiry, not a raffle close date. Stopgap: store it in metadata and run your own scheduler to stop sales + draw. Whop will not auto-close it.
await whop.plans.update(ticket.id, {
  metadata: { raffle: "true", raffle_ends_at: "2026-07-01T00:00:00Z" },
});
// Proper fix → ends_at on a real /raffles resource (see "What's missing").

7 Send link to product raffle

const checkout = await whop.checkoutConfigurations.create({
  company_id: company.id,
  plan_id: ticket.id,
});
// → checkout.purchase_url  → share this in the group chat / socials
// (or share ticket.purchase_url from step 5 directly)

After the drop

Beyond the seller journey, but needed to actually get paid and deliver the prize.

// Payouts — KYC + bank so ticket revenue can land
const link = await whop.accountLinks.create({
  company_id: company.id,
  use_case: "account_onboarding",
  refresh_url: "https://jacks-raffles.com/refresh",
  return_url: "https://jacks-raffles.com/done",
});

// Track entries (webhook on payment.succeeded = new ticket)
await whop.webhooks.create({
  company_id: company.id,
  url: "https://jacks-raffles.com/webhooks",
});

// After drawing a winner, ship the prize + attach tracking
await whop.shipments.create({
  company_id: company.id,
  payment_id: "pay_winner_xxx",
  tracking_code: "1Z999AA10123456784",
});

What's missing needs new endpoints

The journey works end-to-end on existing APIs except steps 6–7's raffle semantics. A raffle isn't a first-class object today — it's approximated by a plan + metadata + manual drawing. Real gaps:

// 1. The raffle primitive — collapses steps 3–7, owns deadline + ticketing
//    → POST /raffles
//      ({ company_id, item_template, image_id, ticket_price, currency,
//         total_tickets, max_tickets_per_user, ends_at, winner_count, draw_mode }
//       → { id: "raf_xxx", checkout_url, status })

// 2. Buy N tickets in one purchase (plans only do 1 fixed-price entry)
//    → POST /raffles/{id}/tickets   ({ quantity } → entries + payment)

// 3. Auto-close at the deadline (step 6 has no real backing today)
//    → ends_at on /raffles + a worker that closes & draws at the deadline

// 4. Pick the winner — provably fair (RNG seed + audit record)
//    → POST /raffles/{id}/draw   ({ winner_count } → winners[], proof)

// 5. Entries / odds transparency
//    → GET /raffles/{id}/entries   (entrants + ticket counts)

// 6. AI design generation (step 4's "prompt to design product page")
//    → POST /designs   ({ prompt, reference_file_ids[] } → file_ids)
Not an endpoint, but a blocker: the matching business classification sweepstakes_raffles is NON_DISCOVERABLE + FINANCING_RESTRICTED (hidden from Discover, payout limits), and raffles are legally regulated — loop in compliance before launch.
Built on Whop's existing /api/v1 public API. Steps 1–4 and 7 ship as-is; steps 5–6 are approximations; winner-draw and true ticketing need new endpoints.
