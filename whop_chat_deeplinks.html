Android testing (develop build)

Build: com.whop.whopapp.develop  ·  signed in as @deeppandya91.

Browser tap. Android only auto-verifies the /joined/* path, so a tapped whop.com/messages… link opens the browser, not the app. To make taps open the app, either prefix the path with /joined/ (the parser strips it, same destination — e.g. whop.com/joined/messages?chat=…), or enable it once in Settings → Apps → Whop (develop) → Open by default → Open supported links → add whop.com.

adb (always works, bypasses link verification). Pushes the URL as the deeplink extra. The URL is single-quoted inside the device command so an unquoted & isn't eaten by the phone's shell (which would truncate the link before &message=… and skip the scroll):

adb shell "am start -n com.whop.whopapp.develop/com.whop.whopapp.MainActivity \
  --es deeplink '<URL>'"

Android-specific behavior notes

  • Deeplinks delivered while the app is backgrounded/cold are replayed once the app is ready (fixed: were dropped on a replay-0 event bus before the UI subscribed).
  • Scroll-to-message loads a window around the target by id if it's not already paged in, then centers & briefly highlights it.
  • A deeplink launch suppresses the townhall feed load and the rating prompt, so it opens straight to the target.
  • AI chat (/chats/<id>) has no native screen → opens in a WebView.
